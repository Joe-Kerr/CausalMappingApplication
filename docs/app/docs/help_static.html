Add a set

To add a set to a statement right click on it and the context menu appears. Click on an empty space in the bottom most box. If a drop down list appears containing the desired set, just click on it. If the desired set is not present, notice the blinking cursor which indicates that you can enter the new set. Hit enter after you have typed in the set.

You have another option to add new sets to the map but not to a specific statement. To do so click on the menu "Settings"->"Manage sets". Click in the top left box and enter the new set and confirm with enter.

If you have selected multiple statements, the behaviour of adding sets becomes relative. The initial selection is empty. All sets you then add will be assigned to the selected statements. This difference in behaviour is rooted in a conflict of managing statement sets if individual statements have different sets. So, if you have statements "A" and "B" with sets "alpha" and "beta" respectively, which sets to display in a multi-selection of "A" and "B"? The union would show "alpha" and "beta". What if you wanted to add "beta"? The relative approach ensures that adding sets to multiple statements works as expected.

Notice that set names must begin with an alphabetic character and contain only alphanumeric characters or whitespaces for technical reasons.

Add and manage views

Views are like sets for visual purposes. You can assign views to statements and choose to display statements of a particular view only. This makes working with very large maps easier, e.g. by hiding long cross links. When working with views you may notice greyed-out statements. These are the direct neighbours of non-active views and their purpose is to not break the context of the active view statements. This is, of course, still just a compromise since their context in turn is lost. Do keep this in mind when working with views.

There always is a default view "View 1". To add a view click the drop down field and select "New". You now can switch to the new view by clicking on the arrow keys or by selecting it in the drop down field. Each statement will be assigned to the currently active view. You can change views for individual or multiple statements via their context menu.

Right click on the drop down field for further options. You can toggle whether to show all statements and you can also create sets from your views. The view-all mode is cancelled whenever you switch a view.

If you do not have a need for sets, keep selected the default value.

Conduct analyses

Some of the available analyses restrict the interactivity of the map.

Analyses are bound to the state of the map data at the time of starting a respective analysis. In principle, if you change the data while an analysis is running, you will invalidate the analysis without corrective measures. Similarly, if you work with analysis results, but change the underlying data, any data displayed that is based on analysis results, will be invalid. Therefore, the corrective measure to be taken would have to be to restart the analysis.

This may not always be desirable. On a large map, for instance, a cluster analysis may take a few seconds. You would quickly run out of patience if every change event on the map triggered a restart of the analysis. Or, think about a situation where you accidentally change your map yet base conclusions on the analysis data. The consequences may be catastrophic.

Therefore, most of the analyses automatically restrict the interactivity of the map after you have started them. Interactivity is restored once all analysis windows are closed. You can manually switch back the edit or link mode at your own risk.

Purpose

Particularly for legal or scientific application you may require a rigorous audit trail of your mapping sessions.

Remember that each cognitive map is a snapshot or an (intermediary) end-product. It does not show how it came about. One way to show this historic process is a log.

Usage

The activity log is operated with the start and stop buttons.

You can choose to continue logging even if the activity log window is closed. This way it will not occupy work space while you are mapping. Do not forget to save the log once you are done with your session.

You can also choose to list all screen elements that existed before you started the logging process. This is important if you continue working on an existing map and ensures that an inspector does see a complete log.

Central analysis

Purpose

A central analysis seeks to find important statements in a map and hence central issues. Importance is measured by the number of "neighbours", i.e. links in from, and out to, other statements. The central analysis considers the neighbours of the neighbours up to a desired depth (i.e. the neighbours of the neighbours of the neighbours...etc.) and adds the numbers to a total score per statement. The higher the number of neighbours the greater the importance. The most central statements can then be used for a summary of the map or can be fed to other analyses that require some seed data.

It can be argued that the central analysis is better capable to deal with outliers than the domain analysis. That is because in a domain analysis a statement with many neighbours gets a high score. But if this statement's neighbours only have a relatively small number of neighbours themselves, the statement may not be that important as its domain score suggests. Since the central analysis measures neighbours' neighbours it can even out these "local hot spots".

Depth of neighbours

There is no rule concerning the right depth of neighbours. Looking at different depth settings is a good way forward. Care should be taken on maps that have many paths with varying depths. If there are only few paths with high depth whereas the rest has substantial lower depths, the high depth paths' scores keep growing and lowering the relative scores of the low depth paths. Just imagine a path consisting of three statements A->B->C. The maximum depth, of this path, is reached after layer 2 for statement A. That means the score of A will not grow beyond layer 2. Now oppose a chain from some statement D to another statement K. This chain will grow beyond layer 3.

Distance weighting

A decay function can be applied to counteract inflation of importance. That means you might want to assign diminishing weight to more distant neighbours.

Usage

The central analysis window provides you the number of neighbours' neighbours per each statement. Depths between 0 and 5 are displayed. A decay can be set to put less weight on neighbours that are further away from the statement in focus. The decay function accounts for distance by division by the reciprocal of the respective order of neighbours. The raw data can be copied to your clipboard for use in external programmes such as Microsoft Excel.

Visual aids

You also get two visual aids. A histogram shows you the distribution of scores by percentile ranks.

On the map itself, statements are coloured as a heatmap. The colouring is temporary and lasts until you close the analysis window. The heatmap is used as a way to visualise a range of numbers. It starts from blue over green, yellow, orange and ends at red. This colour gradient matches the number range in ascending order (blue lowest numbers, red highest numbers). For technical reasons central scores are normalised and put into a range of 255.

Sources

Eden, C., Ackermann, F. and Cropper, S. (1992), "The Analysis of Cause Maps", Journal of Management Studies, Vol. 29 No. 3, pp. 309-324.

Reynolds, M. and Holwell. S. (2010) "Systems Approaches to Managing Change: A Practical Guide", Springer, London.

Cluster analysis

Purpose

Cluster analysis can be described as a way to put together elements into groups. Usually, the grouping is done by similarity, i.e. if elements are similar to each other, they will be assigned to the same cluster.

In reality, some authors claim that we do not even know when a cluster a cluster is. Some other authors say that cluster analysis is not just a kind of analysis but a way of knowing. Consider this as an indicator of the breadth of the field of cluster analysis.

For the present purposes, cluster analysis is taken as technique to group statements into sets. This process is data-driven. That means the data determines, through a measure, the grouping. This is unsupervised as opposed to, for instance, when you group statements into custom sets yourself which is supervised. The advantage is that the grouping is more authentic or closer to the actual data. The groups emerge from the data rather than being fit into the data.

But what are these groups good for? These groups or clusters contain statements more closely related to one another than to the rest of the map. In other words they indicate themes in a map. These themes are ideal for finding key issues that may be relatively more important.

Usage

Currently, clustering is possible by Jaccard index and hierarchy drill down.

Jaccard index

The Jaccard index measures similarity by comparing statements pairwise for how many neighbours (in and out linking statements) they have in common. The standard threshold is 0. That means similarity just has to be greater than 0%. Put differently pairs are considered not similar if they do not share any neighbour. Naturally, this analysis will find all distinct clusters in a map that have no links to any other cluster. Increasing the threshold will gradually rip apart the clusters. Clusters that stay together longer are stronger. The maximum setting is 1 (0.999...) meaning that statement pairs must share the same neighbours. Since this is more often than not the case, the analysis will form many clusters of size 1.

Notice that the algorithm assigns each statement only once (clusters are mutually exclusive) and that equal similarity scores of two or more statements are resolved in a first come first serve manner.

The analysis window provides you an input box for setting a custom threshold. On larger maps the analysis may take a couple of seconds. The analysis results are presented in tabular form as well as visually on the map by giving clusters of statements distinct colours. The colouring is temporary and will be removed once you close the analysis window. You can also assign each cluster to a set.

Hierarchy drill down

The clustering by hierarchy takes an initial group of statements (seed set). For each seed statement the entire chain of in-linking statements is assigned to a cluster (including the seed statement). That means for every initial statement the chain of argument is drilled down. Phrased in terms of similarity, all statements belong to a cluster if they are descendants of a seed statement.

Statements can occur in multiple clusters so that clusters are not mutually exclusive.

By default, all head statements are preselected as seed statements. You can disable this option. Additionally, you can specify a custom group of statements to be used as seed statements. On larger maps the analysis may take a couple of seconds. The analysis results are presented in tabular form as well as visually on the map by giving clusters of statements distinct colours. If a statement is a member of multiple clusters, the statement will be coloured respectively. If a custom seed statement is a member of another cluster, it will not be multi-coloured but retain its distinct single colour and get a red outline. The colouring is temporary and will be removed once you close the analysis window. You can also assign each cluster to a set.

Sources

Reynolds, M. and Holwell. S. (2010) "Systems Approaches to Managing Change: A Practical Guide", Springer, London.

Tan, P.-N., Steinbach, M. and Kumar, V. (2005), "Introduction to Data Mining", Pearson, USA.

Uprichard, E. (2009) "7. Introducing Cluster Analysis: What Can It Teach Us about the Case?" in Byrne, D. and Ragin, C.C. (eds), The SAGE Handbook of Case-Based Methods, SAGE Publications Ltd, London.

Context viewer

Purpose

One of the most important principles of Causal Cognitive Mapping, which also goes back to its theoretical roots in Personal Construct Theory, is that the meaning of a statement is implied by its neighbours. Consider the statement "buy a new computer". The meaning of it likely differs depending on whether you considered it in relation to "new technology on the market" or "loading a document takes too long".

Therefore, a facilitative feature is the context viewer. It extracts a statement in focus into a new screen. This is very helpful for "busy" maps with many cross links on the screen or statements far apart.

Uses

Use cases range from simple isolated inspection to extracting a certain statement and its neighbours for presentation purposes. You could for instance extract the top ranking statements by central score as a map summary.

Usage

In the context viewer window the middle statement is the statement in focus. Above it, you find the neighbours the statement in focus links to whereas at the bottom are all the ones that link into it. With the control buttons you can browse through the statements. Alternatively, just click on any of the statement shown in order to jump to it.

Sources

Reynolds, M. and Holwell. S. (2010) "Systems Approaches to Managing Change: A Practical Guide", Springer, London.

Domain analysis

Purpose

A domain analysis counts how many links go in and out per each statement. The higher a statement's count the more important the statement has to be.

It can be said that the domain analysis only detects local importance. A statement with many neighbours but whose neighbours have not many neighbours themselves may not have that much of a weight in relation to the whole map.

Nonetheless, the biggest advantage of the domain analysis is that it can be conducted on paper/whiteboard if need be. On a computer, the domain analysis is rather useful for providing input data to other analyses such as cluster analysis. A central analysis is likely a better choice for finding important statements as it takes a holistic view of importance.

Usage

The domain analysis window provides you the number of neighbours per each statement.

You also get two visual aids. A histogram shows you the distribution of scores by percentile ranks.

On the map itself, statements are coloured as a heatmap. The colouring is temporary and lasts until you close the analysis window. The heatmap is used as a way to visualise a range of numbers. It starts from blue over green, yellow, orange and ends at red. This colour gradient matches the number range in ascending order (blue lowest numbers, red highest numbers). For technical reasons central scores are normalised and put into a range of 255.

Feedback loop analysis

Purpose

Feedback loops rank among the most important phenomena in science. Feedback loops are implied when talking about exponential growth or decay, engineering certain control "systems" or simply when talking with somebody or turning on the hot water for your bath. In general, one calls something a feedback loop if some initial thing causes one or more things that impact back on the initial thing.

Examples and types

Considering the example of exponential growth, that what grew grows itself something which grows itself something (interest, animals, humans)... This is called positive feedback. Something reinforces itself.

If you turn on the hot water flow for your bath, initially, you sense it as not so hot, but then quickly as hot ("Ouch, too hot"). The delayed temperature information causes you to turn up the cold flow. The new delayed temperature information causes you to turn the hot flow up again ("Hooo, too cold"). This goes on for a bit ("Better, but still a little too hot/cold") until you reach a satisfactory temperature. That is an example for negative feedback. Something balances itself.

Consequences

An undesirable consequence of a positive feedback loop can be that something slips out of control whereas in case of a negative feedback loop something gets stuck in the act of balancing. Therefore, you want to detect feedback loops and rigorously investigate them.

In cognitive / cause maps

In a cognitive map or cause map a feedback loop is a chain of statements that starts at statement A and comes back to that statement A. You cannot readily read off the map whether it is a positive or negative loop. This is always contextual.

Usage

In the Causal Cognitive Mapping Application, a feedback loop is detected through translating a map into all possible paths of statements. During that process whenever the next path segment to be added is already contained within the path constructed up to that point the loop is extracted.

The window of the feedback loop analysis outputs the progress of the analysis. You might not recognise this on small maps where the analysis does not take much time.

Once the analysis is done, all feedback loops are displayed. You can click on each statement number to jump to that statement on the map. You can assign the feedback loops to sets using the control button.

Limitations

Think of a map in terms of paths. So, if there is a triangle pattern where A links to B and C, you have two paths: A->B and A->C. The algorithm used moves from possible path to path. For each path being assembled, path segments need to be saved to memory. Each path can yield a feedback loop which is then saved to memory as well.

Therefore, in principle, if you have a large enough map with a large enough number of feedback loops, your computer might run out of memory.

In practice, though, this is unlikely. The World3 model converted to a cognitive map consists of 307 statements and 473 links. The model is based on several causal loop diagrams and is highly inter-connected. It holds 86 feedback loops. The analysis takes a bit over 0.5 seconds and consumes around 7 megabytes of RAM.

Sources

Reynolds, M. and Holwell. S. (2010) "Systems Approaches to Managing Change: A Practical Guide", Springer, London.

Map history

Purpose

The drawback of a causal cognitive map is its static nature. The map, i.e. the output of causal cognitive mapping, is essentially an image. A single image can only ever capture one instance in time. That means the history of the entire mapping session is lost.

The map history mitigates this drawback by creating a video of when in time statements and arrows were added to the map.

Usage

The map history consists of two parts:

  • on map interaction,
  • file export.

The on map interaction is controlled via the start and stop buttons. This will run a live video of the map creation process. With the "follow mode" you can make the screen jump to the element that is being added. The time between steps controls when the next element from the timeline is added.

Both options can be changed while the live video is running.

The file export function lets you save the map history to an animated gif file for use in web presentations or other interactive media.

The file will be saved to the image directory in the application's files directory. If a file with the same name already exists and you did not tick the overwrite checkbox, a counter number is appended to the file.

The animation speed can be controlled through the "Delta time" input box.

Statement browser

Usage

This is an advanced search and filter tool.

You can display statements that contain or do not contain certain keywords. Currently displayed statements can be selected on the map. Clicking on a statement moves the screen to that statement on the map. The "auto"-checkbox, if enabled, will automatically add your phrase to the filter list.

You can use regular expressions as filter phrase. Regular expressions need to be enclosed in dashes: /^your regex$/

If you are unfamiliar with regular expressions, please see the introductory material on developer.mozilla.org or wikipedia. Some useful expressions may be:

  • /^word/: Matches all statements that begin with "word".
  • /word$/: Matches all statements that end with "word".
  • /key|word/: Matches all statements that contain either "key" or "word".

Set analysis

Purpose

Sets, in Causal Cognitive Mapping, are categories of statements. You can either define them by yourself or obtain sets from analyses such cluster analysis.

Sets are a powerful way to summarise your map.

Usage

In the set summary window you first can choose which sets you wish to summarise.

After that you get a summary in matrix and tabular form. Both show cross links between sets. The matrix view puts the sets in relation to each other. If a cell is coloured, it means that the set on the left links to the sets on top. If you hover over the coloured square, the number of links is shown. The colours follow a heat map gradient. Blue represents the relative lowest number of cross links. The further the colour gradient moves toward red a relative higher number of cross links is shown. The tabular view shows essentially the same data but in tabular form.

Signal words analysis

Purpose

Cognitive maps and cause maps are not just directed graphs, and therefore suitable for mathematical analyses, but each statement contains text. Hence linguistic analyses can be applied as well.

Two simple yet powerful analysis techniques search for prescriptive words and "glue" words.

Prescriptive words such as must, ought or should are best to be avoided altogether in Causal Cognitive Mapping. These words may hinder map development since they imply lack of alternatives. If you for instance assume that you must do "A", naturally you may not consider doing "B". A positive side effect is that you save words since instead of "must do A" you can just write "do A".

Glue words, such as in order to, due to, through or because, are not bad per se. They may indicate though that your single statement may actually be expressed as two statements. That is because glue words merge two (main/sub-) clauses causally. If you say for example "I always do A because of B", then you can map this as B->A.

Usage

In the analysis window you get a list of statements. You can click on a statement number to jump to that statement on the map.

Sources

Reynolds, M. and Holwell. S. (2010) "Systems Approaches to Managing Change: A Practical Guide", Springer, London.

Similarity analysis keywords

Purpose

The analysis for similarity discussed here is about text similarity of statement pairs for the purpose of finding undiscovered links or statements that can be merged. Both text similarity and other similarity types (e.g. shared neighbours) are also integral part of cluster analysis.

Usage

The keyword analysis first finds all words within the statements that occur at least twice. After that, statements are listed that contain a keyword. A dual listing is presented: on top you can scroll through statement pairs whereas at the bottom you can view a list of the matches.

With the settings tab you can fine tune the analyses. The stopwords filter skips common English words such as "the", "a", "he" "do", etc. in the analysis operation. Similarity analysis metric

Purpose

The analysis for similarity discussed here is about text similarity of statement pairs for the purpose of finding undiscovered links or statements that can be merged. Both text similarity and other similarity types (e.g. shared neighbours) are also integral part of cluster analysis.

Usage

Using the Jaccard index, statement pairs are displayed that are x-% similar. Similarity refers to how many words they have in common.

With the settings tab you can fine tune the analyses. The Jaccard threshold option sets that statement pairs must be at least that similar in order to be displayed. The stopwords filter skips common English words such as "the", "a", "he" "do", etc. in the analysis operation. Since purpose of the similarity analysis here is to find links or mergers, you can opt to not display statements that are already linked.

Notice: a computer does not know the concept of a word. E.g. this "random line of text", is considered as one unit or string. Therefore, some (internal only) changes during the calculation are made:

  • all characters that are not alphanumeric or white spaces are replaced by white spaces
  • consecutive white spaces are collapsed
  • white spaces become word boundaries

One positive side effect is that phrases like "apples/peaches" or "problem-solving" become actually two words.

Summary statistics

Overall statistics

Number of statements / links

These statistics are strongly correlated with how the underlying map data were obtained. If it were primary data, you probably have gained them by interviewing. Factors such as duration or structure of the interview impact on the total numbers of statements/links rather than the "cognitive complexity" of the interviewee. If you obtained your data from text sources, investigate their contexts. How were they obtained and by whom? Do they summarise? Do they elaborate? Again, do not conclude about cognitive complexity without careful analysis.

Links per statement

Although there is no empirical basis, mapping guidelines based on practical experience suggest that a usual ratio is between 1.15 and 1.2. Deviations may just be rooted in the nature of the map. But they may also indicate improvement potential. A high ratio may suggest that you have redundant links. It suffices for instance if you map: A->B->C. There is no need for linking A->C since this is implicit in A->B->C.
However, this is only valid for cognitive maps of individuals. In a cause map composed of many individual cognitive maps, different people may draw different paths of causality. Whether redundant links are in fact redundant depends on the agreement of the map stakeholders.

Head and tail statements

Head statements are those with only statements linking in to them. Tail statements are those with only statements linking out of them.

Unlinked statements

Statement that do not link to and do not receive links from any other statement. The presence of unlinked statements would be unusual hence worth a look.

Heads per statements

The ratio tells you how many goals or values you have per statement.

Tails per statements

The ratio tells you how many options you have per statement.

Statement statistics

#

The statement number.

Txt

The text content of the statement.

Set/s

The set/s of the statement, if any.

Role

  • Transmitter: Statement has no connections coming in to itself but sends connections out.
  • Receiver: Statement has connections coming in to itself but does not send connections out.
  • Ordinary: Statement has connections coming in to itself and sends connections out.
  • Isolated: Statement has no connections coming in to itself and does not send connections out.

In-degree and Out-degree

In-degree is the number of statements that link into the statement in focus whereas out-degree is the number of statements it links to.

Usage

The analysis window displays various data.

Sources

Eden, C., Ackermann, F. and Cropper, S. (1992), "The Analysis of Cause Maps", Journal of Management Studies, Vol. 29 No. 3, pp. 309�324.

Word count analysis

Purpose

Among the strengths of Causal Cognitive Mapping is its capacity to summarise a complex issue. But even if that was not of concern to you, statements that contain too many words make working with the map tedious.

Therefore, the recommended word count per each statement lies between 8 and 12 depending on mapping guidelines.

Usage

In the analysis window you can set a desired word count threshold. If the word count of a statement is higher than the threshold, it will be listed. You can choose to double the threshold for statements that contain a bipolar contrast.

Sources

Reynolds, M. and Holwell. S. (2010) "Systems Approaches to Managing Change: A Practical Guide", Springer, London.
